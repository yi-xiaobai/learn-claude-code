# 工作日志管理 Skill

**目标文件**: `$WORK_LOG_PATH`，使用前加载配置：

```bash
source ~/.claude/skills/work-log/.env
```

## 四类操作

- **A. 添加/更新任务**：把任务写到指定日期的 `### YYYY.MM.DD` 标题下，标题不存在则按时间顺序插入。
- **B. 记录出差/标注日期状态**：出差优先写成日期区间标题和事项；请假、半天假、休假、团建、补班等状态写到日期标题备注中，标题不存在则创建日期标题。
- **C. 生成未来一周日期模板**：在文件末尾追加未来若干天的工作日标题（需调用节假日 API `http://api.haoshenqi.top/holiday?date=YYYY-MM-DD`，跳过周末和假日，标注补班）。
- **D. 导出 PDF**：执行 `bash ~/.claude/skills/work-log/scripts/export-pdf.sh`（依赖 MWeb Pro + 终端辅助功能权限）。

## 请求解析

一次请求可组合多个操作。先识别操作指令，再把剩余文本作为任务内容，**不要把操作指令当任务**。

**触发 D（导出 PDF）的关键词**：`导出pdf` / `导出 PDF` / `输出 pdf` / `生成 pdf` / `export pdf`，以及作为动作短语出现的 `pdf`（如句末 `& pdf`）。识别后剥离该片段及其前后分隔符（`&`、`并`、`然后`、`，` 等）再解析任务。

**触发 C**：`生成模板` / `下周模板` / `未来一周模板` / `生成日期`。

**触发 B（出差/日期状态）的关键词**：`出差` / `差旅` / `外出办公` / `客户现场` → 出差记录；`请假` / `半天假` / `休假` / `团建` / `补班` 按原词标注。出差如果包含日期区间（如 `5.11~5.15`、`5.11-5.15`、`5月11日到5月15日`），写成区间标题，剩余文本作为出差事项；如果没有区间，则按单日记录。

**歧义**：操作关键词被引号 `""`、书名号 `「」`、括号包裹时按任务内容处理。

**示例**：
- `昨天完成了 X & 导出pdf` → A + D
- `5.11~5.15 出差处理电信问题 & 导出pdf` → B + D，标题为 `### YYYY.05.11~05.15（出差）`，事项为 `处理电信问题`
- `今天出差处理电信问题` → B，标题为 `### YYYY.MM.DD（出差）`，事项为 `处理电信问题`
- `明天去上海出差处理客户现场问题` → B，标题为 `### YYYY.MM.DD（出差：上海）`，事项为 `处理客户现场问题`
- `生成下周模板并导出pdf` → C + D
- `今天完成了「导出pdf功能」` → 仅 A

## 任务格式

```markdown
### YYYY.MM.DD~MM.DD（出差）

- [x] 出差事项
- [ ] 未完成的任务
  - [x] 已完成的子任务
```

- 状态判定：`完成了` / `已完成` / `做完了` → `[x]`；`在做` / `进行中` / `待处理` → `[ ]`
- 出差区间格式：`### YYYY.MM.DD~MM.DD（出差）`；跨月写 `### YYYY.MM.DD~MM.DD（出差）`，跨年写完整年份；出差带地点时用 `### YYYY.MM.DD~MM.DD（出差：地点）`
- 出差事项写在标题下方任务列表中，如 `- [x] 处理电信问题`
- 单日日期可加备注：`### YYYY.MM.DD（请假）`；多状态用顿号连接，如 `### YYYY.MM.DD（请假、补班）`
- 常见标记：出差、请假、半天假、休假、团建、补班
- 任务可含 Jira 链接：`[CZ-xxx](url)`

## 约束

- 新日期按时间顺序插入；遵循现有文件格式
- 标注出差时优先创建或更新出差区间标题，不把“出差”单独写成任务；若同句还有实际工作内容，作为出差事项追加
- 已存在同一区间出差标题时只追加事项，避免重复创建标题
- 已存在单日日期标题时保留原备注并补充新状态，避免重复标记；例如已有 `（补班）`，再标请假则改为 `（请假、补班）`
- 同一日期只追加，不删除或修改已有任务
- 添加前检查日期是否已存在，避免重复
