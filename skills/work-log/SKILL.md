# 工作日志管理 Skill

**目标文件**: `$WORK_LOG_PATH`，使用前加载配置：

```bash
source ~/.claude/skills/work-log/.env
```

## 三类操作

- **A. 添加/更新任务**：把任务写到指定日期的 `### YYYY.MM.DD` 标题下，标题不存在则按时间顺序插入。
- **B. 生成未来一周日期模板**：在文件末尾追加未来若干天的工作日标题（需调用节假日 API `http://api.haoshenqi.top/holiday?date=YYYY-MM-DD`，跳过周末和假日，标注补班）。
- **C. 导出 PDF**：执行 `bash ~/.claude/skills/work-log/scripts/export-pdf.sh`（依赖 MWeb Pro + 终端辅助功能权限）。

## 请求解析

一次请求可组合多个操作。先识别操作指令，再把剩余文本作为任务内容，**不要把操作指令当任务**。

**触发 C（导出 PDF）的关键词**：`导出pdf` / `导出 PDF` / `输出 pdf` / `生成 pdf` / `export pdf`，以及作为动作短语出现的 `pdf`（如句末 `& pdf`）。识别后剥离该片段及其前后分隔符（`&`、`并`、`然后`、`，` 等）再解析任务。

**触发 B**：`生成模板` / `下周模板` / `未来一周模板` / `生成日期`。

**歧义**：操作关键词被引号 `""`、书名号 `「」`、括号包裹时按任务内容处理。

**示例**：
- `昨天完成了 X & 导出pdf` → A + C
- `生成下周模板并导出pdf` → B + C
- `今天完成了「导出pdf功能」` → 仅 A

## 任务格式

```markdown
### YYYY.MM.DD

- [x] 已完成的任务
- [ ] 未完成的任务
  - [x] 已完成的子任务
```

- 状态判定：`完成了` / `已完成` / `做完了` → `[x]`；`在做` / `进行中` / `待处理` → `[ ]`
- 日期可加备注：`### YYYY.MM.DD（请假）`，常见标记：请假、半天假、休假、团建、补班
- 任务可含 Jira 链接：`[CZ-xxx](url)`

## 约束

- 新日期按时间顺序插入；遵循现有文件格式
- 同一日期只追加，不删除或修改已有任务
- 添加前检查日期是否已存在，避免重复
